# Services Directory

Contains service classes that implement application business logic.

## Purpose

- Orchestrate domain objects and business operations
- Handle complex business workflows
- Manage transactions and data consistency
- Coordinate between different parts of the application

## Current Services

### UserService

- User management operations
- CRUD operations for users
- User search and filtering
- Password management

### AuthService

- Authentication logic
- Token management
- User authorization
- Session handling

## Best Practices

- Follow single responsibility principle
- Use dependency injection
- Implement proper error handling
- Add comprehensive unit tests
- Document public methods
